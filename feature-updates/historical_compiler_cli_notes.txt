# Epic Request: Historical Compiler CLI Arguments and E2E Mode

**Context**: This is one of 7 runner script refactoring features originally designed in
KAI-10 (architectural_refactoring_configuration_management) but never implemented. The
full specification was completed in KAI-10 S2 and can be referenced directly.

## Background

compile_historical_data.py already has 4 args (--year, --verbose/-v, --enable-log-file,
--output-dir), but is missing the universal args (--e2e-test, --log-level) and two
script-specific args for network constants currently in historical_data_compiler/constants.py:
- REQUEST_TIMEOUT = 30.0 (line 98)
- RATE_LIMIT_DELAY = 0.3 (line 101)

These two constants need to move from constants.py to argparse defaults.

The BaseHTTPClient already accepts timeout and rate_limit_delay via constructor params
(historical_data_compiler/http_client.py lines 62-82) — no refactoring needed there.

## E2E Mode

**Purpose:** The --e2e-test flag enables a repeatable, automated end-to-end scenario designed
for CI/CD integration testing. The integration test framework (feature 8 of this series) calls
each script with --e2e-test via subprocess to validate the full pipeline end-to-end without
needing human interaction. This is distinct from unit tests — it exercises the real network
call, real data parsing, and real file I/O through the actual CLI entry point.

**Requirements for a valid E2E mode:**
- Completes in ≤180 seconds (network latency must be within this budget)
- Writes all output to /tmp (never to simulation/sim_data/) — /tmp is outside the repo and never committed
- Exits 0 on success; exits 0 with info message if required data files are missing (graceful skip)
- Deterministic: same command always tests the same thing (1 season, 100-player limit)

**Behavior:** fetches 1 season with player_limit=100. Uses tempfile.TemporaryDirectory() for
output — temp dir is auto-cleaned after the run. Never writes to simulation/sim_data/.

## Goals

1. Add universal args: --e2e-test, --log-level
2. Add script-specific args: --timeout, --rate-limit-delay
3. Remove REQUEST_TIMEOUT and RATE_LIMIT_DELAY from historical_data_compiler/constants.py
4. Wire timeout/rate_limit_delay from CLI through compile_season_data() to BaseHTTPClient()
5. Implement --e2e-test mode: 1 season, player_limit=100, output to system temp dir (not sim_data/)
6. Preserve --verbose flag backward compatibility (maps to DEBUG logging)

## Design Decisions (from KAI-10 S2)

- No --debug flag: --e2e-test serves both purposes
- Direct params pattern (no @dataclass Settings): values flow through compile_season_data() signature
- E2E data limiting: pass player_limit=100 to fetch_player_data() (add param to that function)
- E2E output: use tempfile.TemporaryDirectory() — temp dir auto-cleaned, never touches sim_data/
- --verbose preserved (pre-existing backward compat): equivalent to --log-level DEBUG
- Additional args (--weeks, --validate, --clean, --skip-backups) are out of scope

## Files to Modify (4 files)

1. compile_historical_data.py — Add 4 new CLI args, wire to BaseHTTPClient, E2E temp dir logic
