## Request: Fix player_fetcher E2E output to use fixed path

**Created:** 2026-02-19
**Discovered during:** KAI-11 S2 checklist resolution (Q7)
**Type:** Bug fix (KAI-10 post-release correction)

---

## Problem

`run_player_fetcher.py` currently uses `tempfile.mkdtemp()` for E2E output:

```python
# run_player_fetcher.py lines 159-163
if args.e2e_test:
    tmp_dir = tempfile.mkdtemp(prefix='player_fetcher_e2e_')
    position_json_output = str(Path(tmp_dir) / 'player_data')
    team_data_folder = str(Path(tmp_dir) / 'team_data')
    game_data_csv = str(Path(tmp_dir) / 'game_data.csv')
```

User preference: **always use fixed paths** for E2E output. Random tmpdirs make smoke
tests harder to verify (can't predict output path without capturing stdout).

---

## Desired Behavior

Replace random tmpdir with fixed paths:

```python
if args.e2e_test:
    position_json_output = '/tmp/player_fetcher_e2e/player_data'
    team_data_folder = '/tmp/player_fetcher_e2e/team_data'
    game_data_csv = '/tmp/player_fetcher_e2e/game_data.csv'
```

Also update the corresponding test in `tests/root_scripts/test_root_scripts.py`
(TestRunPlayerFetcher) and any smoke test documentation that references the old path.

---

## Scope

- Modify: `run_player_fetcher.py` (replace tempfile.mkdtemp with fixed /tmp path)
- Update: any tests or docs referencing the old tmpdir pattern

---

## Design Constraint

**Always use fixed paths for E2E output** — applies to ALL runner scripts, not just
player_fetcher. This is a global preference established 2026-02-19.
