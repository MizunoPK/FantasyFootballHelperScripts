## Purpose

This framework is the CI/CD integration layer for the entire runner script series. Each runner
script's --e2e-test flag exists specifically to enable this framework — the flag provides a
deterministic, ≤180s, zero-human-interaction entry point that these integration tests invoke
via subprocess.

The integration tests validate the full pipeline end-to-end: CLI argument parsing, business
logic, real network calls (where applicable), and file I/O — all through the actual CLI entry
point. This is distinct from unit tests, which mock dependencies and test components in isolation.

**CI/CD usage:** Run `python run_all_integration_tests.py` in a CI pipeline after deploying
changes to any runner script. Exit code 0 = all 7 scripts working end-to-end. Exit code non-zero
= at least one script has a regression.

## Background

After all 7 runner scripts are refactored with --e2e-test support, this feature creates
an automated CLI integration test framework that validates each script end-to-end using
subprocess invocation. Tests assert exit codes and specific outcomes (not just that the
script ran — but that it produced expected output).

## Test Pattern (for all 7 test runners)

```python
result = subprocess.run(
    [sys.executable, str(project_root / "run_SCRIPT.py"), "--e2e-test"],
    capture_output=True, text=True, timeout=180
)
assert result.returncode == 0
assert "Traceback" not in result.stderr
```

Timeout handling: tests warn but do not fail on >180s (network variability).
3-5 argument combinations tested per script.
